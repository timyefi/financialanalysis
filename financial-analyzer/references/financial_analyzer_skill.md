# 财务分析技能 - 详细指南

## 概述

将公司年报/半年报/评级报告PDF转换为投行标准Excel财务分析工作底稿，并在底稿基础上产出正式研报。

分析维度与 Excel 正式化已拆成独立约束文件：

- `references/analysis_dimension_contract.md`
- `references/excel_formalization_contract.md`

## 报告写作标准

最终输出不要写成摘要型说明，而要写成资深固收/信用分析师的研究稿。建议先完成标准化 Excel 工作底稿，再基于底稿写报告。建议结构：

1. 基本信息
2. 结论先行
3. 重大风险发现
4. 按附注科目逐章分析
5. 关键计算表
6. 学习点 / 信用判断 / 最终结论

写作时优先展开，并尽量按财报科目命名小节：

- 货币资金 / 受限资金
- 应收账款 / 其他应收款
- 存货 / 资本化利息 / 抵押与担保
- 短期借款 / 长期借款 / 应付债券 / 租赁负债
- 财务费用 / 利息资本化
- 资产减值 / 信用减值
- 现金流量表补充资料
- 外汇风险 / 诉讼 / 或有事项 / 契约

要求：

- 每个关键判断都要有原文证据和数字
- 必须给出派生比率或同比变化
- 允许表格，鼓励表格
- 禁止空泛套话和模板摘要
- 结论必须透明可追溯，读者能顺着证据回到原文
- `chapter_records.jsonl` 和逐章阅读 ledger 是主写作底座；`final_data.json` / `soul_export_payload.json` 只能做索引与交叉校验
- 如果证据索引太稀疏，先回到章节级抽取，不要直接写报告
- 风格可参考 `cases/碧桂园2024年年报分析.md` 与 `cases/中海地产2024年年报分析.md`

## 知识库动态刷新

- 在计算任何指标、阈值或派生比率前，先查 `financial-analyzer/knowledge_base.json` 和最近的 adoption logs，优先使用知识库中的公式口径。
- 如果知识库暂时没有覆盖到某个稳定口径，先用章节证据和显式公式完成可验证计算，不要把新经验写死在 skill 文本里。
- 在章节复核结束后，把新发现的公式、例外规则、阈值和表述方式先写入 `financial-analyzer/knowledge_base.json`，再通过 adoption log 让后续案例复用。

## 使用流程

### 步骤1：使用MinerU技能解析PDF

调用MinerU技能将PDF解析为Markdown格式：
- 输入：PDF文件路径
- 输出：MinerU输出目录（包含markdown文件）

### 步骤2：提取财务数据（资深研究员阅读法）

**核心原则**：像资深研究员一样阅读财报——先定位章节，再逐页阅读，严禁一次性全量读取。

#### 2.1 定位关键章节（搜索+定位）

使用 `search_content` 工具快速定位章节位置，确定开始阅读的行号：

```
搜索目标：
- "六、合并财务报表项目注释" 或 "合并财务报表项目注释"
- 确认行号（如：第1326行开始）
```

**优化策略**：
- 通过搜索定位，将开始阅读的行数尽量往后移
- 优先搜索"附注"、"财务报表项目注释"等关键词
- 确认行号后再开始逐页阅读，减少无效阅读量

**推荐搜索关键词**（按优先级排序）：
1. "六、合并财务报表项目注释" - 最精确定位
2. "合并财务报表项目注释" - 快速定位
3. "五、合并财务报表" - 如有单独财务报表章节
4. "附注" 或 "财务报表附注" - 通用的附注开始位置

#### 2.2 逐页阅读附注（资深研究员法）

使用 `read_file` 工具**逐段**读取，每次限制行数（建议60-80行）：

```
阅读规则：
1. 先用搜索定位章节起始行号
2. 从起始行开始，每次读取60-80行
3. 阅读完成后根据内容判断是否继续
4. 遇到债务相关科目（18-23号）重点关注
```

#### 2.3 提取资产负债表数据

**重要**：有息债务的详细构成在附注章节中！资产负债表只显示汇总数，明细在"合并财务报表项目注释"中。

**附注章节阅读流程**：

1. **搜索定位**：使用 `search_content` 搜索"合并财务报表项目注释"
2. **确认行号**：从搜索结果确认起始行号（如：第1326行）
3. **逐页阅读**：从起始行开始，每次读取60-80行
4. **识别科目编号**：附注按# 18、# 19、# 20...编号

**债务相关附注科目（重点）**：

| 科目编号 | 科目名称 | 关键信息 | 有息认定 |
|---------|---------|---------|---------|
| # 18 | 其他应付款 | 含政府借款、往来款 | 仅关联方/政府借款算 |
| # 19 | 一年内到期的非流动负债 | 一年内到期长借、债券、长期应付款 | ✅ 全部算 |
| # 20 | 其他流动负债 | 短期融资券 | 融资券算，待转销项税不算 |
| # 21 | 长期借款 | 保证/抵押/质押借款明细 | ✅ 全部算 |
| # 22 | 应付债券 | 债券名称、发行规模、利率 | ✅ 全部算 |
| # 23 | 长期应付款 | 融资租赁、信托借款 | 需识别具体性质 |

**附注明细提取要点**：

1. **# 18 其他应付款**：
   - 必须看"主要明细"表格
   - 识别"款项的性质"列：借款→有息，保证金/押金→无息

2. **# 19 一年内到期非流动负债**：
   - 直接使用合计数（全部为有息债务）
   - 包含：一年内到期长借、一年内到期债券、一年内到期长期应付款

3. **# 20 其他流动负债**：
   - 短期融资券→有息
   - 待转销项税→无息

4. **# 21 长期借款**：
   - 看"保证借款/抵押借款"明细
   - 扣除一年内到期部分后为净余额

5. **# 22 应付债券**：
   - 查看债券明细表（名称、发行金额、利率）
   - 扣除一年内到期部分

6. **# 23 长期应付款**（关键！）：
   - 必须看明细表中每笔的"性质"
   - 融资租赁、保理、信托 → ✅ 有息
   - 往来款、保证金 → ❌ 无息

请从MinerU输出的markdown文件中，提取以下资产负债表科目数据：

| 科目中文名 | 科目英文名 | 提取要点 |
|-----------|-----------|---------|
| 货币资金 | Cash and cash equivalents | 关注受限货币资金 |
| 应收账款 | Accounts receivable | 关注账龄、坏账准备 |
| 存货 | Inventories | 房地产关注在建开发产品 |
| 固定资产 | Fixed assets | 关注折旧方法 |
| 在建工程 | Construction in progress | 资本化利息 |
| 短期借款 | Short-term borrowings | 关注担保方式 |
| 长期借款 | Long-term borrowings | 关注到期分布 |
| 应付债券 | Bonds payable | 关注期限、利率 |
| 一年内到期非流动负债 | Non-current liabilities due within one year | 重点关注 |
| 负债合计 | Total liabilities | 交叉验证 |
| 所有者权益合计 | Total owners' equity | 关注资本公积 |


#### 2.4 提取利润表数据

| 科目中文名 | 科目英文名 | 提取要点 |
|-----------|-----------|---------|
| 营业收入 | Operating revenue | 确认时点 |
| 营业成本 | Operating cost | 与收入配比 |
| 销售费用 | Selling expenses | 占比分析 |
| 管理费用 | Administrative expenses | 关注折旧摊销 |
| 财务费用 | Financial expenses | 利息支出单独列示 |
| 利息支出 | Interest expense | **关键指标** |
| 净利润 | Net profit | 扣除非经常性损益 |

#### 2.5 提取现金流量表数据

| 科目中文名 | 科目英文名 | 提取要点 |
|-----------|-----------|---------|
| 经营活动产生的现金流量净额 | Net cash from operating | **核心指标** |
| 销售商品、提供劳务收到的现金 | Cash received from sales | 收现比计算 |
| 购建固定资产、无形资产和其他长期资产支付的现金 | Capital expenditures (CAPEX) | 资本支出 |
| 固定资产折旧 | Depreciation | EBITDA计算 |
| 无形资产摊销 | Amortization | EBITDA计算 |

- 主体评级（ issuer rating）
- 评级机构
- 评级展望
- 本次评级变动

### 步骤3：计算财务指标

基于提取的财务数据，计算以下核心指标：

#### 偿债能力指标
- 流动比率 = 流动资产 / 流动负债
- 速动比率 = (流动资产 - 存货) / 流动负债
- 资产负债率 = 负债合计 / 资产总计
- 产权比率 = 负债合计 / 所有者权益

#### 有息债务指标（核心）

**重要**：有息债务必须从附注明细中逐项识别，不能简单使用资产负债表汇总数！

**有息债务认定标准**：

| 类型 | 是否算有息 | 判断依据 |
|------|-----------|---------|
| 银行长期借款 | ✅ 算 | 银行借款，付息 |
| 关联方/政府借款 | ✅ 算 | 承担利息成本 |
| 应付债券 | ✅ 算 | 债券融资 |
| 短期融资券 | ✅ 算 | 债券融资 |
| 融资租赁 | ✅ 算 | 实质是债务融资 |
| 信托/保理融资 | ✅ 算 | 实质是债务融资 |
| 一年内到期非流动负债 | ✅ 算 | 包含长借、债券、长期应付款 |
| 应付票据 | ❌ 不算 | 跟随业务现金流 |
| 上下游/供应商往来款 | ❌ 不算 | 经营性质，不付息 |
| 押金/保证金 | ❌ 不算 | 经营性质 |

**有息负债计算公式**：
```
有息负债 = 长期借款 + 应付债券 + 短期融资券 + 一年内到期非流动负债 + 长期应付款(融资租赁) + 关联方借款
```

**净负债** = 有息负债 - 货币资金
**净负债率** = 净负债 / 所有者权益

#### 盈利能力指标
- 毛利率 = (营业收入 - 营业成本) / 营业收入
- 净利率 = 净利润 / 营业收入
- ROE = 净利润 / 所有者权益

#### EBITDA相关指标（核心）
- EBITDA = 净利润 + 所得税 + 利息支出 + 折旧 + 摊销
- 利息保障倍数(ICR) = EBITDA / 利息支出
- 净负债/EBITDA

#### 现金流指标
- 经营现金流/债务 = 经营活动现金流 / 有息负债
- 自由现金流(FCF) = 经营现金流 - 资本支出
- 净现比 = 经营现金流 / 净利润
- 收现比 = 销售收现 / 营业收入

### 步骤3.5：生成 Excel 工作底稿

在开始写 Excel 之前，先把逐章阅读得到的内容整理成“发现池”，再按主题归类，最后才写临时脚本。不要把模板脚本产物直接当成正式 Excel。

#### 3.5.0 先读两个约束文件

- 先按 `analysis_dimension_contract.md` 检查本案必须覆盖的维度。
- 再按 `excel_formalization_contract.md` 检查 workbook 的结构、顺序和展示要求。
- 如果维度覆盖不完整，禁止进入落表阶段。

#### 3.5.1 整理发现池

发现池至少应包含：
- 章节原文证据
- 附注抽取数据
- 需要调用的知识库公式口径
- 需要计算的派生指标
- 风险信号与管理层支持信号
- 待归类事项和待确认口径

#### 3.5.2 归类发现

把发现按以下维度整理后再决定 workbook 结构：
- 总览
- 关键指标
- 债务结构
- 流动性与契约
- 资产质量
- 现金流桥梁
- 附注映射
- 管理层支持
- 资本结构
- 证据索引

#### 3.5.3 编写一次性临时脚本

临时脚本必须使用刚才抽取和加工过的数据以及公式，生成保留公式关系的 `financial_output_formula.xlsx`。公式版应体现完整 workpaper 逻辑，不能只填静态数值。

#### 3.5.4 先公式版，再最终版

在核对公式版的口径、公式关系和 sheet 组织后，再由公式版派生最终 `financial_output.xlsx`。正式版生成后要复核是否存在空值、错位、单位混写、来源链路丢失或公式被硬编码替代。

#### 3.5.5 推荐的 workbook 结构

正式 Excel 的 sheet 组合可以参考以下结构，具体可随案例调整，但功能面不能缺：
- `00_overview`
- `01_kpi_dashboard`
- `02_financial_summary`
- `03_debt_profile`
- `04_liquidity_and_covenants`
- `05_asset_quality`
- `06_cash_flow_bridge`
- `07_notes_map`
- `08_management_support`
- `09_capital_structure`
- `99_evidence_index`

#### 3.5.6 结构设计原则

- `00_overview` 必须保留发现来源、回退下载、MinerU 解析、formalization 路径和人工复核链路。
- `07_notes_map` 负责把附注章节、行范围、主题标签和摘要对上，作为正式底稿与报告的桥梁。
- `99_evidence_index` 必须标签化，不能直接堆整段章节原文。
- 公式版 workbook 要保留派生过程，最终版 workbook 再固化展示结果。

## 注意事项

1. **数据单位**：以原始披露和工作底稿口径为准，常见统一单位为亿元；如需转换，必须在输入台账中显式说明
2. **数值提取**：优先提取带有千分位逗号的数值
3. **多期间数据**：如有多期数据，建议分别提取后汇总分析
4. **异常值检测**：对明显异常的数据（如负数、零值）需人工复核
5. **债务明细必须从附注提取**：
   - 资产负债表只有汇总数
   - 必须阅读"合并财务报表项目注释"章节
   - 必须逐项识别债务性质（有息 vs 无息）
6. **长期应付款特别关注**：
   - 融资租赁/保理/信托 = 有息
   - 上下游往来款 = 无息
7. **货币资金受限情况**：
   - 必须从附注中提取受限比例
   - 受限资金不能用于偿债

## 阅读效率优化原则

1. **搜索优先**：先用 `search_content` 定位关键章节行号，再开始阅读
2. **逐段限制**：每次 `read_file` 不超过80行，避免上下文溢出
3. **识别边界**：当看到"第七章"或"备查文件"时，表示附注章节已结束
4. **债务科目重点**：# 18-23号附注是债务明细核心，无需阅读全部34个科目
5. **明细vs汇总**：
   - 资产负债表 = 汇总数
   - 附注 = 明细构成
   - 有息债务必须从附注明细中逐项识别
6. **定性优先**：
   - 先判断债务性质（有息/无息）
   - 再提取金额数字
   - 避免将往来款、保证金误算为有息负债

## 自我进化机制

### 每次实践后的总结流程

每次完成一个公司的财报分析后，必须执行以下总结：

#### 步骤1：记录新发现

使用 `update_memory` 工具记录本次实践中获得的新知识。

#### 步骤2：更新Skill文档

将重要新发现添加到 `financial_analyzer_skill.md` 中。

#### 步骤3：典型案例归档

将典型案例记录到 `cases/` 目录。

### 需要记录的重点

1. **特殊债务类型**：首次遇到的债务类型及认定
2. **附注格式变化**：附注表格的特殊格式处理
3. **数据提取技巧**：高效提取数据的提示
4. **易错点**：容易误判的情况及纠正
5. **行业特性**：特定行业的债务特点

### 进化触发条件

满足以下任一条件时，必须触发进化：

| 触发条件 | 动作 |
|---------|------|
| 发现新的债务类型 | 更新"有息债务认定标准" |
| 遇到新的附注格式 | 更新"附注定式识别" |
| 踩坑并解决 | 更新"注意事项" |
| 发现高效新方法 | 更新"阅读效率优化原则" |
| 完成N个案例 | 复盘并提炼通用规则 |
