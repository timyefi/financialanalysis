# Excel 正式化约束

## 1. 目的

把“怎么生成 Excel”单独拆成正式化约束，避免财务分析、知识提炼和工作簿落表互相缠在一起。

Excel 正式化只负责把已经完成的分析结果结构化成工作簿，不负责替代逐章阅读和知识采纳。

## 2. 输入边界

Excel 正式化的输入只能来自以下产物：

- `chapter_records.jsonl`
- `chapter_review_ledger.jsonl`
- `notes_workfile.json`
- 经过复核的知识库口径
- 已确认的 discovery pool / focus list

不能直接拿 scaffold 文件当成正式 workbook 的唯一输入。

## 3. 强制顺序

所有正式 Excel 生成必须遵循：

1. 先形成发现池。
2. 再按维度归类。
3. 再在当前 `run_dir` 内编写一次性临时脚本。
4. 先生成 `financial_output_formula.xlsx`。
5. 复核公式、口径、sheet 组织和证据链。
6. 再由公式版固化出 `financial_output.xlsx`。
7. 最后才做视觉微调或兼容输出。

## 4. 结构要求

如果是标准年报/半年报分析，工作簿至少要覆盖这些功能面：

- 总览
- KPI 摘要
- 财务摘要
- 债务结构
- 流动性与契约
- 资产质量
- 现金流桥
- 附注映射
- 管理层支持
- 资本结构
- 证据索引

sheet 名称可以因案例微调，但功能面不能缺失。

## 5. 展示要求

- 正式汇总区默认统一用亿元口径。
- 证据索引必须标签化，不能整段堆原文。
- 概览页必须保留来源链路、下载/解析/复核路径。
- 公式版必须保留关键勾稽关系，不能在第一版就全部硬编码成值。
- 正式版必须检查空值、错位、单位混写和证据丢失。

## 6. 复核标准

完成一个 workbook 时至少复核：

1. sheet 数量和命名是否满足案例目标。
2. 公式版与最终版是否存在一致的计算口径。
3. 数值是否统一到同一展示单位。
4. 证据索引是否能回到章节级证据。
5. 概览页是否保留分析来源链路。

## 7. 与分析流程的关系

Excel 正式化是分析流程的后半段，不是分析流程本身。

也就是说：

- 先完成章节阅读、知识比对和维度覆盖。
- 再做 Excel 正式化。
- 不能先做模板，再倒逼分析。
