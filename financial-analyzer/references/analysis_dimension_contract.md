# 财务分析维度约束

## 1. 目的

把“财务分析必须覆盖哪些维度”单独抽出来，避免在执行时只做章节摘要、只做指标计算，或者只按个案习惯选择少数专题。

该约束的来源有两部分：

- `target_case/vanke_2025_semiannual_20260406` 里的样本 workbook 维度。
- `financial-analyzer/knowledge_base.json` 里的稳定分析维度、公式口径和风险信号。

## 2. Target Case 维度

对照 `vanke_2025_semiannual_20260406` 的目标 workbook，标准分析维度必须覆盖：

1. `00_overview`：基本信息、结论先行、来源链路、复核路径。
2. `01_kpi_dashboard`：收入、利润、现金、债务、存货等关键指标摘要。
3. `02_financial_summary`：资产负债表、利润表、现金流量表的合并摘要。
4. `03_debt_profile`：短债、长债、债券、租赁负债、净债务。
5. `04_liquidity_and_covenants`：现金、受限资金、短债覆盖、支持项、契约/到期压力。
6. `05_asset_quality`：存货、其他应收款、合同资产、投资性房地产、减值。
7. `06_cash_flow_bridge`：利润到现金的桥梁、资本化利息、筹资依赖。
8. `07_notes_map`：章节到表格的附注映射与证据回溯。
9. `08_management_support`：股东支持、公开债务偿还、再融资窗口、持续经营措辞。
10. `09_capital_structure`：总债务、净债务、权益、杠杆结构。
11. `99_evidence_index`：标签化证据索引，不能直接堆原文。

## 3. 知识库维度

`knowledge_base.json` 里的分析维度不是单纯指标列表，而是可复用的判断框架。执行分析时必须显式覆盖以下维度：

- 偿债能力：流动比率、速动比率、现金比率、资产负债率、产权比率、现金短债比。
- 有息债务：长期借款、应付债券、短期融资券、融资租赁、信托/保理、关联方借款、一年内到期非流动负债。
- 盈利能力：毛利率、净利率、ROE、ROA、ROIC、EBITDA 利润率。
- 现金流：经营现金流/债务、自由现金流、净现比、收现比、利息保障倍数。
- 资产质量：存货质量、应收账款坏账、其他应收款沉淀、合同资产信用风险、投资性房地产、公允价值、抵押/担保。
- 风险信号：受限资金、重大担保、或有事项、资本化利息、减值、持续经营、再融资依赖、管理层支持。
- 行业修正：房企、LGFV、港股准则、境外债、多币种、政府补助等专题口径。

## 4. 强制执行方式

每次分析必须先构建“维度覆盖矩阵”，把章节和指标映射到下列大类：

- 总览与基本信息
- 利润与盈利质量
- 现金流与短债
- 债务与资本结构
- 存货与资产质量
- 受限资金、担保和或有事项
- 管理层支持与持续经营
- 专题修正项

如果某个大类没有证据，就必须在 `chapter_review_ledger.jsonl` 里明确写成“未抽取到/未识别”，不能跳过。

## 5. 交付要求

1. `chapter_review_ledger.jsonl` 必须能回溯到维度层，而不是只回溯到章节层。
2. `financial_output.xlsx` 必须覆盖至少一张总览、一张债务、一张流动性、一张资产质量、一张现金流桥、一张证据索引。
3. 若案例存在行业专题，专题必须进入独立模块或独立行区，不得混入总览摘要替代。
4. 维度覆盖矩阵属于强制中间产物，可放在 `focus_list_scaffold.json` 或章节复核记录中，不能省略。
