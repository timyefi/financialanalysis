# Soul 结构化主表 Excel 规范（v1）

## 1. 定位

Soul 是对外输出的标准化成品，服务于：

- 信用分析师/基金经理的单主体深度研究
- 跨主体横向比较
- 跨年度纵向跟踪

Soul 不是内部训练中间产物；内部进化数据应写入 `pending_updates.json`、`knowledge_base` 等内部文件。

## 2. 设计原则

1. **专业性**：金融、咨询行业可直接使用。
2. **美观性**：统一视觉层级、可读性强、适合汇报。
3. **标准性**：核心字段固定、口径一致。
4. **开放性**：允许行业扩展字段，不破坏主结构。
5. **可追溯**：关键指标可追溯到原文证据。

## 3. Sheet 结构建议

## 3.1 `00_cover`（封面与目录）
- 发行主体
- 报告期
- 数据更新时间
- 编制说明与口径摘要

## 3.2 `01_entity_profile`（主体画像）
- 公司全称
- 主体评级/展望
- 核心业务与区域
- 股权结构要点
- 核心信用观点摘要

## 3.3 `02_financial_core`（财务主表）
- 资产负债表核心字段（多期）
- 利润表核心字段（多期）
- 现金流核心字段（多期）
- 单位统一与口径备注

## 3.4 `03_debt_notes_detail`（附注债务明细）
- 债务类型
- 本期余额
- 上期余额
- 是否有息
- 期限结构
- 融资工具属性
- 证据索引（章节/段落）

## 3.5 `04_rating_insights`（评级视角）
- 评级结论
- 关键支撑因素
- 关键约束因素
- 展望与触发条件
- 与财务指标交叉验证结论

## 3.6 `05_kpi_panel`（指标看板）
- 杠杆：净负债率、总债务/EBITDA
- 偿债：ICR、经营现金流/有息负债
- 盈利：毛利率、净利率、ROE
- 现金流：FCF、净现比
- 同比变化和阈值提示

## 3.7 `06_peer_comparison`（同业对比）
- 同期同业样本列表
- 核心指标并列比较
- 分位点/排名
- 差异点评

## 3.8 `99_evidence_index`（证据索引）
- 字段名
- 字段值
- 来源文档（年报/评级）
- 来源定位（页码、章节、行号）
- 解析置信度

## 4. 核心字段与扩展字段机制

- **核心字段（固定）**：跨主体必须存在，保证可比性。
- **扩展字段（可选）**：按行业补充，命名遵循 `ext_<domain>_<name>`。

示例：
- `ext_property_presale_cash`
- `ext_infra_concession_years`

## 5. 视觉与格式规范（面向专业输出）

1. 主色建议：深蓝 + 灰阶（咨询风格）。
2. 所有 Sheet 使用统一字体、字号、行高、千分位格式。
3. 指标区分：
   - 原始值（黑色）
   - 计算值（深蓝）
   - 风险提示（橙/红）
4. 图表要求：趋势线 + 同业对比柱图，避免花哨装饰。
5. 导出即交付：无需二次人工排版。

## 6. 与技能体系的关系

- 解析能力由 MinerU 提供（PDF -> Markdown/结构化文本）。
- Financial Analysis 负责计算、判断和证据绑定。
- 推荐新增独立“Excel 生成技能”用于专业排版和模板化导出。

## 7. 验收标准（v1）

1. 单主体至少 3 年对比完整。
2. 核心字段缺失率 < 5%。
3. 每个关键结论可追溯至证据索引。
4. 同业对比 Sheet 可直接用于投研讨论。
5. 外部读者无需接触内部中间产物即可理解结论。
